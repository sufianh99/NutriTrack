    **# NutriTrack**
    One-paragraph description: web-based nutrition tracking app, individual daily energy/macro calculation (Mifflin-St-Jeor), food tracking with Soll/Ist comparison and color-coded progress.

    **## Features**
    Bullet list: Profile/Onboarding, BMR/TDEE/Macro calculation, Food entry CRUD, Open Food Facts search, Dashboard with Soll/Ist + Ampel, Date navigation/history.

    **## Architecture**
    - Tech stack: Python 3.9+, Flask 3.1.3, SQLAlchemy 2.x, SQLite, Jinja2, Flask-WTF
    - Project structure overview (tree of key files):
      ```
      NutriTrack/
      ├── app/
      │   ├── __init__.py          # Flask app factory
      │   ├── models.py            # SQLAlchemy models (UserProfile, DailyGoal, FoodEntry)
      │   ├── calculator.py        # BMR, TDEE, macro calculation (pure Python)
      │   ├── nutrition.py         # Portion scaling, daily summation, progress status
      │   ├── api_client.py        # Open Food Facts API wrapper
      │   ├── routes.py            # Flask routes (Blueprint 'main')
      │   ├── forms.py             # WTForms form classes
      │   └── templates/           # Jinja2 templates
      ├── tests/                   # pytest test suite
      ├── config.py                # App configuration
      ├── run.py                   # Development server
      ├── wsgi.py                  # Production WSGI entry point
      └── requirements.txt         # Python dependencies
      ```
    - Two core modules explanation:
      1. Bedarfsberechnung (calculator.py): Mifflin-St-Jeor BMR, TDEE with PAL factors, goal modifiers, macro split (25/30/45)
      2. Ernaehrungstracking (nutrition.py + routes): Food CRUD, portion scaling, daily summing, Soll/Ist comparison with color-coded Ampel
    - Data model: UserProfile (single row, id=1), DailyGoal (one per date), FoodEntry (multiple per date)

    **## Setup (Lokale Installation)**
    Step-by-step:
    1. `git clone` + `cd NutriTrack`
    2. `python -m venv venv` + activate (show Windows + Linux/Mac)
    3. `pip install -r requirements.txt`
    4. `cp .env.example .env` and edit SECRET_KEY
    5. `python run.py` — opens at http://127.0.0.1:5000
    6. First visit redirects to onboarding form

    **## Usage (Bedienungsanleitung)**
    1. Onboarding: Enter age, height, weight, gender, activity level, goal
    2. Dashboard: View calculated Soll values (kcal, protein, fat, carbs)
    3. Add Food: Click "Hinzufuegen", optionally search Open Food Facts, fill form, save
    4. Track Progress: Dashboard shows Ist values, progress bars with Ampel colors (green 90-100%, orange/red >100%)
    5. History: Navigate to past dates via arrow buttons on dashboard

    **## Testing**
    ```bash
    pip install -r requirements-dev.txt  # or: pip install pytest pytest-flask black ruff mypy
    python -m pytest tests/ -v
    python -m black --check .
    python -m ruff check .
    python -m mypy app/ --ignore-missing-imports --disallow-untyped-defs
    ```

    **## CI/CD**
    Brief: GitHub Actions pipeline runs on push/PR — Black, Ruff, Mypy, pytest. Badge if available.

    **## Deployment**
    - Render: Create Web Service, connect repo, set build command `pip install -r requirements.txt`, start command from Procfile, set SECRET_KEY env var
    - Railway: Connect repo, auto-detects Procfile, set SECRET_KEY env var
    - PythonAnywhere: Upload code, set WSGI file to point to wsgi.py, install requirements
